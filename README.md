# 비접촉 입욕 안전 모니터링 시스템

욕조에 넣는 완전 방수 압력 센서로 **정상 입욕과 위험 상황(익수 가능성)을 구분**하는 비접촉 안전 디바이스. 카메라·웨어러블을 쓰지 않는다. **1차 대상은 노인 입욕 침수**(재택·독거 노인의 욕조 익사)이며, 영유아 욕조는 확장 대상으로 둔다.

핵심 기술은 **수중 압력의 시간 패턴(dP)으로 정상 입욕과 침수를 분리**하는 것이다. 정상 출렁임은 baseline으로 복귀(복귀형)하고, 잠김은 레벨이 유지(지속형)된다는 시간 패턴 차이를 이용한다.

> **현재 단계 = Phase A.** "압력 데이터만으로 정상 입욕과 침수가 실제로 분리되는가"를 검증 중이다. 이 검증(진행/피벗 게이트)이 통과되기 전에는 통신·허브·클라우드·앱·양산 하드웨어를 착수하지 않는다.

> 기술적 한계·미검증 항목은 [RISKS.md](./RISKS.md)에 사전 공개한다. 모든 정확도·분해능 수치는 실측 대체 전까지 데이터시트/참고문헌 **인용 기준**이다.

> 압력 센서의 작동 원리(절대압·수위·분해능)는 [SENSOR.md](./SENSOR.md)에 정리한다.

---

## 문제 정의

노인의 욕조 사망은 **"넘어짐(낙상)"이 아니라 "욕조 안에서 잠김(침수)"이 압도적**이다. 입욕 중 저혈압성 실신·심정지 등으로 서서히 물에 잠기는 사건이 주를 이룬다. 상용 비접촉 감지(레이더 등)는 **낙상**을 보지만, **물에 잠긴 상태(침수)**는 보지 못한다. 카메라는 욕실 프라이버시로 불가하다. 수중 압력은 잠긴 신체의 변위를 물리적으로 직접 측정하는 접근이다.

---

## 핵심 설계 원칙

| 원칙 | 내용 |
|---|---|
| 1차 신호 = 수중 압력 | 잠긴 신체 부피 변화 → 수면 변화 → 압력. 카메라·레이더가 못 보는 수중을 물리적으로 직접 측정 |
| 스칼라 dP로는 부족 | 격한 물장구와 침수는 **크기가 겹친다.** 실제 분리축은 **시간 패턴**(복귀형 vs 지속형) |
| 낙상이 아닌 침수 | 상용 레이더는 낙상 감지. 본 시스템은 **물에 잠긴 상태**를 압력으로 감지 — 다른 물리 현상 |
| 생명 알림은 로컬 우선 | 인터넷이 끊겨도 동작해야 한다. 클라우드·앱은 원격 조회 보조, 응급 경보는 로컬 |

## 선행연구 · 차별점

압력 기반 익수 감지는 선행이 존재한다.

| 유형 | 선행 | 본 시스템과의 차이 |
|---|---|---|
| 학술 | **Kharrat et al. (2012)** — 착용형 압력센서와 신경망을 이용하여 near-drowning과 정상 수영을 분류 | 착용형·수영장 대상. 욕조 고정 압력센서 기반 정상 입욕/침수 구분은 다루지 않음 |
| 학술 | **Murao et al.** — 욕조 바닥 수압센서를 이용한 사용자 식별 | 사용자 식별·점유 판단이 목적이며 정상 입욕/침수 분류는 아님 |
| 특허 | **EP1918191A1** — 수압·호흡·자세 등을 이용한 욕조 익사 위험 감지 | 다중 센서 기반. 바닥 압력 변화 패턴 단독 분석과 다름 |
| 특허 | **US20150107015A1** — 영유아 욕조 익사 감지 및 자동 배수 | 모션 기반·영유아 대상. 압력 시간 패턴 방식이 아님 |
| 상용 | **Milesight VS373** | 낙상 감지 또는 웨어러블 방식. 욕조 바닥 압력 기반 비접촉 방식과 다름 |

|---|---|---|
| 규제 적합성 (병렬) | 의료기기 분류 리스크 제거 | 식약처 의료기기 해당여부 **사전질의 1차 답변** 확보 |
| A. 압력 검증 | 구조화 데이터 수집 + 통계 분석 → 판정 | 아래 [Phase A 판정 기준](#phase-a-판정-기준-확정) 충족 + 한계(Risk 01·02·03) 명문화 |
| 진행 / 피벗 게이트 | 진행 또는 피벗 결정 | 진행 = 분리 성공 / 피벗 = 실패 → 역할 재정의(입수·퇴수 검출 + 비정상 패턴) + 2차 모달리티 추가 |

| C. 통신 링크 | 완전 밀폐 하 알림 경로 확보 | 알림 지연 목표 + 배터리 목표 |
| D. 허브 게이트웨이 | 로컬 우선 알림 + 수신/ACK/상태 | 인터넷이 끊겨도 로컬 경보 동작 + 패킷 신뢰성 |
| E. Cloud + 앱 | 원격 조회 + 푸시 | 외부 망에서 상태 조회 + 푸시 도달률 목표 |
| F. PCB · 인증 | 양산 HW + 방수 신뢰성 + 인증 | 장기 방수 신뢰성 통과 + 인증 통과 |

### Phase A 판정 기준 (확정)

충돌쌍(정상 vs 침수가 가장 헷갈리는 쌍: N4-D5, N2-D3, N5-D2)에서 판정한다.

| 게이트 | 지표 | 기준 | 비고 |
|---|---|---|---|
| 신호 존재 | SNR | **≥ 10 dB** | 정의 `20·log10(\|peak dP\| / σ)`, σ는 실측 노이즈로 확정 |
| 분리력 (핵심) | AUC | **≥ 0.90** | 충돌쌍별, 특징 F1~F6 통합 분류기, Leave-One-Trial-Out CV |
| 진단 | Cohen's d | 핵심 특징 **≥ 1.5** | 어떤 특징이 신호를 싣는지 + 과적합 감별 |
| 피벗 조건 | AUC | 어느 충돌쌍이라도 **< 0.75** | 압력 단독 분리 실패 |

> 두 정규분포 가정에서 `AUC = Φ(d/√2)`이므로 d와 AUC는 서로 다른 축(단일 특징 vs 통합 분류기)으로 분리해 사용한다. 원안의 `d≥0.8 AND AUC≥0.9`는 부정합(AUC≥0.9 ⇒ d≈1.81)이라 폐기했다.

## 분석 방법

현재는 **직접 정의한 특징값 기반 통계 분류**로 간다. 시계열 딥러닝은 데이터 부족 시 과적합·설명 불가로 부적합하며, 데이터가 축적된 후로 미룬다.

- 특징 F1~F6: 지속성 · 복귀 여부 · 복귀 시간 · 빈도 · 누적 dP · 단조성
- 공통 처리: 이동평균(평활화) → 1차 미분(변화율) → 임계값 + 지속시간(오탐 억제)

## 실험 대상

**성인 남성 자발적 잠수를 우선**한다. 실제 인체는 부피·호흡·미세움직임을 재현하므로 모형보다 외적 타당성이 높고, 노인 욕조 침수 역학과도 정합한다.

> **안전 전제:** 자발적 잠수도 천수성 실신 위험이 있다. 감시자 상시 동석·얕은 수심·짧은 시간이 필수이며, 단독 실험은 금지한다. (RISKS.md Risk 14)

실제 익사 데이터는 윤리적으로 수집 불가능하므로, Phase A는 "실제 익사 감지"가 아니라 **"침수 변위 패턴의 분리 가능성"**을 검증한다.
