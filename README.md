# 논카메라·논웨어러블 AI 안전 모니터링 디바이스

한 가구 안에서 영유아부터 노인까지의 안전을 **단일 시스템**으로 보호하는 비접촉 AI 안전 알림 디바이스. 거실·안방의 메인 허브와 욕실의 워치독이 연동되어, 카메라를 사용하지 않고 호흡·움직임·낙상·체온 변화·수면 변화를 감지해 보호자에게 즉시 알림.

> **기술적 한계 및 검증 계획 [RISKS.md](./RISKS.md)** — 핵심 가설과 미검증 항목을 사전에 공개한다.

## 기술 핵심

| 구분 | 내용 |
|---|---|
| 포지셔닝 | 안전 알림 디바이스 — 비의료기기 포지셔닝 목표 (식약처 사전 질의 예정) |
| 센서 융합 | 60GHz mmWave + 수중압력(MS5803-14BA) + IR Thermal + PIR |
| 아키텍처 | 메인 허브(거실·안방) + 워치독(욕실), ESP-NOW 연동 |
| 핵심 설계 | 영유아 욕조: **압력센서 1차 트리거 → 60GHz 2차 교차검증**의 4단계 직렬 필터 |
| 학습 구조 | 공개 데이터셋 사전학습 → 베타 fine-tuning → 연합학습 (raw 미전송) |

> **설계 원칙:** 60GHz는 물에 흡수되어 잠긴 순간 신호가 급감하므로, 욕조 익사를 단독 감지하는 메인 센서가 아님. 물리적으로 가장 확실한 1차 신호는 수중 압력이며, 60GHz는 이를 교차검증하는 역할로 재정의함.

## 현재 진행 상태
설계 후 직접 학습하며 구현을 진행 중.

| 영역 | 상태 |
|---|---|
| 시스템·센서 아키텍처 설계 | 완료 |
| 감지 카테고리 등급 분류 (A/B/C) | 완료 (일부 항목은 PHASE A 검증 후 확정) |
| 카테고리별 판정 알고리즘 설계 | 완료 (의사코드 수준) |
| 데이터 수집 전략·학습 구조 | 완료 |
| 압력센서 단독 구현 (Phase A) | 진행 예정 |
| 센서 융합 프로토타입 (Phase B~C) | 계획 |
| 베타 검증 | 미착수 |

> 설계서의 정확도 수치 및 변수값은 인용 논문 기준 또는 베타 검증 전 설계 파라미터이며, 자체 검증값이 아님을 명시함.

## 구현 학습 로드맵

직접 구현 역량 확보를 위한 단계별 학습 경로.

- **Phase A — 압력센서 단독 구현**: I2C 통신, ESP32 개발환경, 기본 신호처리(이동평균·변화율·임계값). MS5803-14BA로 수면 급변 검출 최소 구현
- **Phase B — 센서 통합**: ESP32 + ESPHome으로 60GHz mmWave 모듈 + 압력센서 동시 동작
- **Phase C — 융합 판정**: 4단계 직렬 필터 규칙 기반 구현 → 베타 데이터 확보 후 학습 모델 전환

## 차별화

1. **조명 일체형** — 설치 거부감 제거
2. **영유아 욕조 비접촉 감지** — 확인된 범위내 글로벌 상용 사례 없음
3. **자녀 ↔ 부모 양방향 연결** — 경쟁사 전원 단방향 알림
