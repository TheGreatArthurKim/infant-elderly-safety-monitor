SECTION 01
제품 정의 및 현재 우선순위
Product Definition & Current Priority

본 제품은 욕조에 넣는 완전 방수 압력 센서로 정상 입욕과 위험 상황(익수 가능성)을 구분해 보호자에게 알리는 비접촉 안전 디바이스다. 카메라·웨어러블을 쓰지 않는다. 영유아 욕조 익사와 노약자 입욕 사고를 1차 대상으로 한다. 핵심 기술은 수중 압력의 시간 패턴(dP)으로 정상 입욕과 익수를 분리하는 것이다.

CURRENT PRIORITY
현재 단계는 Phase A다. "압력 데이터만으로 정상 입욕과 익수가 실제로 분리되는가"를 검증한다. 이 검증(진행/피벗 게이트)이 통과되기 전에는 Sub-GHz 통신·메인 허브·클라우드·앱·양산 하드웨어를 착수하지 않는다. 기술 리스크를 가장 먼저 제거하는 것이 최우선이다.

설계 원칙

원칙내용
1차 신호 = 수중 압력잠긴 신체 부피 변화 → 수면 변화 → 압력. 카메라·레이더가 못 보는 수중을 물리적으로 직접 측정한다.
스칼라 dP로는 부족격한 물장구와 익수는 크기가 겹친다. 실제 분리축은 시간 패턴(복귀형 vs 지속형)이다.
생명 알림은 로컬 우선인터넷이 끊겨도 동작해야 한다. 클라우드·앱은 원격 조회 보조, 응급 경보는 로컬(허브 자체)이 1차다.
가역성검증 단계 — 단일 센서. 비가역 하드웨어 처리(실리콘 밀봉 등)를 금지한다.

설계서의 정확도·분해능 수치는 실측 대체 전까지 데이터시트/참고문헌 인용 기준이며 자체 검증값이 아니다. 한계·미검증 항목은 RISKS.md에 사전 공개한다.

SECTION 02
압력 검출 원리 및 Phase A 검증 설계
Pressure Detection Principle & Phase A Validation
CORE PRINCIPLE — 복귀형 vs 지속형
정상 목욕의 출렁임은 일시적이며 dP가 곧 baseline으로 복귀한다(복귀형). 익수 상황의 잠김은 수위가 한 단계 올라간 뒤 지속된다(지속형, 레벨 시프트 유지). 스칼라 dP의 크기만으로는 둘이 겹치므로, 분리는 dP의 시간 전개 패턴에서 이뤄진다.

패턴 특징 F1~F6

특징정의분리 근거
F1 지속성dP가 임계 위에 머무는 시간지속형은 길고 복귀형은 짧다
F2 복귀baseline 복귀 여부출렁임은 복귀, 잠김은 미복귀
F3 복귀 시간피크에서 baseline까지 시간복귀형의 핵심 지표
F4 빈도FFT/PSD 주파수 성분출렁임은 주기적 피크, 잠김 정착은 저주파·비주기 — 핵심 분리축
F5 누적 dP구간 dP 적분지속형이 크게 누적
F6 단조성상승 후 유지의 단조 정도잠김은 단조적 레벨 시프트

시나리오 및 반복

정상 N1~N6, 익수 D1~D5를 정의한다. 정상과 익수가 가장 헷갈리는 충돌쌍(N4-D5, N2-D3, N5-D2)이 분리 가능성의 실제 시험대다. 일반 시나리오는 시나리오당 30회 이상, 충돌쌍은 50회 이상 반복한다. 약 360회 시점에 중간 판정을 두어 가망이 없으면 조기에 피벗한다.

분석 방법

특징별 분리 크기는 Cohen's d(진단), 통합 분류 성능은 ROC/AUC(핵심 게이트), 신호 품질은 SNR로 본다. F4 빈도는 FFT/PSD로 추출해 출렁임(주기적)과 잠김(저주파·비주기)을 가른다. 분류기는 F1~F6 기반 로지스틱 회귀·결정트리를 쓰고, 소표본 과적합은 Leave-One-Trial-Out CV로 막는다. 표본이 작아 딥러닝은 쓰지 않는다(과적합·과대 해석 우려).

Phase A 판정 기준 (확정)

게이트지표기준비고
신호 존재SNR≥ 10 dB20·log10(|peak dP| / σ), σ=0.5mbar → peak dP ≥ 1.58mbar
분리력 (핵심)AUC≥ 0.90충돌쌍별, F1~F6 통합 분류기, Leave-One-Trial-Out CV
진단Cohen's d핵심특징 ≥ 1.5어떤 특징이 신호를 싣는지 + 과적합 감별
피벗 조건AUC어느 쌍 < 0.75압력 단독 분리 실패 → 역할 재정의 + 2차 모달리티

두 정규분포 가정에서 AUC = Φ(d/√2)이므로 AUC≥0.9는 d≈1.81에 해당한다. 원안의 "d≥0.8 AND AUC≥0.9"는 부정합(느슨+빡셈 AND)이라 폐기하고, d는 단일특징 진단·AUC는 통합 게이트로 축을 분리했다.

SECTION 04
상용 V1 시스템 아키텍처
Commercial V1 System Architecture

최종 제품은 방수 센서 모듈과 메인 허브로 구성된다. 고객이 외출·해외에서도 조회할 수 있도록 허브는 클라우드에 연결되고 앱은 클라우드를 통해 조회한다. 단, 생명 알림은 인터넷에 의존시키지 않는다.

# 통신 경로 방수 센서 모듈(ESP32 + MS5803 + 배터리 + Sub-GHz) │ Sub-GHz ▼ 메인 허브(게이트웨이) ── 로컬 우선 경보: 사이렌 · LED · LAN 푸시 │ 인터넷(Wi-Fi) ▼ Cloud ──▶ 스마트폰 앱(원격 조회 · 이력)
LOCAL-FIRST ALERT (RISK 13)
응급 경보를 클라우드 경로에만 의존하면 인터넷 장애 시 알림이 실패해 생명 위험으로 직결된다. 따라서 응급 경보는 허브 자체 경보 + 동일 LAN 푸시가 1차이고, 클라우드·원격 앱은 보조(원격 조회)다. 센서↔허브 Sub-GHz 링크 신뢰성(ACK/Retry)이 안전의 핵심이다.

메인 허브 역할 및 개발자 모드

허브는 게이트웨이로서 Sub-GHz 수신, 데이터 무결성(CRC), 패킷 재전송(ACK/Retry), 센서·배터리 상태 관리, 클라우드·앱 연동, OTA를 담당한다. 동시에 개발용으로 PC 실시간 확인·CSV·디버깅을 제공하는 개발자 모드를 둔다. 출력 대상(클라우드·로컬 대시보드·CSV·시리얼)을 싱크 인터페이스로 추상화하면 싱크 추가가 재설계 없이 가능하다.

60GHz는 욕조 익사를 단독 감지하는 메인 센서가 아니라, 압력이 포착한 사건을 "사람인가 / 호흡이 없는가"로 교차검증하는 역할이다.

감지 카테고리 및 데이터 전략

6개 카테고리(낙상·욕조·호흡·무움직임·체온·패턴)로 확장하고, 통합 응급도 점수로 일반/주의/위급을 분기한다. 학습은 공개 데이터셋 사전학습 → 베타 fine-tuning → 연합학습(raw 미전송)의 3단계로 진행하며, 영유아 학습 목표는 "익사 분류"가 아닌 "정상 입욕 One-Class 이상 감지"로 정의한다. 모든 정확도 수치는 인용 기준이며 베타 실측 검증 전 가설이다.
