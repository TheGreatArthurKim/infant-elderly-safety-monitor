# 비접촉 입욕 안전 모니터링 시스템

욕조 내부에 설치한 완전 방수 압력 센서를 이용하여 **정상 입욕과 침수(익수 가능성)를 구분**하는 비접촉 입욕 안전 모니터링 시스템이다. 카메라·웨어러블을 쓰지 않는다. **1차 대상은 노인 입욕 침수**(재택·독거 노인의 욕조 익사)이며, 영유아 욕조는 확장 대상으로 둔다.

핵심 원리는 **수중 압력의 시간 패턴(dP)으로 정상 입욕과 침수를 분리**하는 것이다. 정상 입욕에서 발생하는 압력 변화는 기준 압력(baseline)으로 복귀하는 반면,
침수는 압력 변화가 일정 시간 지속되는 특성을 이용한다.

### 한눈에 보기

| 항목 | 내용 |
|---|---|
| 무엇 | 욕조 방수 압력센서로 정상 입욕과 침수를 dP 시간 패턴으로 구분하는 비접촉 안전 디바이스 |
| 1차 대상 | 노인 입욕 침수 (영유아 욕조는 확장 대상) |
| 현재 단계 | **Phase A — 연구 검증** (제품 개발 이전 단계) |
| 검증 질문 | 압력 데이터만으로 정상 입욕과 침수가 분리되는가 |
| 판정 (확정) | 충돌쌍 AUC ≥ 0.90 → 진행 / AUC < 0.75 → 피벗 |

> 기술적 한계·미검증 항목은 [RISKS.md](./RISKS.md)에 사전 공개한다. 모든 정확도·분해능 수치는 실측 대체 전까지 데이터시트/참고문헌 **인용 기준**이다.
>
> 압력 센서의 작동 원리(절대압·수위·분해능)는 [SENSOR.md](./SENSOR.md)에 정리한다.

## 연구 가설 및 현재 단계 (Phase A)

압력 데이터만으로 정상 입욕 상태와 침수 상태를 신뢰성 있게 구분할 수 있는가.

현재는 제품 개발이 아니라 이 가설을 검증하는 **연구 검증 단계(Phase A)**다. 이 검증(진행/피벗 게이트)이 통과되기 전에는 통신·허브·Cloud·앱·양산 Hardware를 착수하지 않는다. AI·딥러닝·Cloud·앱은 모두 이 가설이 성립한 이후의 확장 요소이며, 현재 범위가 아니다.

---

## 문제 정의

노인의 욕조 사망은 "넘어짐(낙상)"이 아니라 "욕조 안에서 잠김(침수)". 입욕 중 저혈압성 실신·심정지 등으로 서서히 물에 잠기는 사건이 주를 이룬다. 상용 비접촉 감지(레이더 등)는 **낙상**을 보지만 **물에 잠긴 상태(침수)**는 보지 못하고, 카메라는 욕실 프라이버시로 쓸 수 없다. 수중 압력은 잠긴 신체의 변위를 물리적으로 직접 측정하는 접근이다.

---

## 핵심 설계 원칙

| 원칙 | 내용 |
|---|---|
| 1차 신호 = 수중 압력 | 잠긴 신체 부피 변화 → 수면 변화 → 압력. 카메라·레이더가 못 보는 수중을 물리적으로 직접 측정 |
| 압력 변화량(dP)의 크기만으로는 부족 | 격한 물장구와 침수는 **크기가 겹친다.** 실제 분리축은 **시간 패턴**(복귀형 vs 지속형) |
| 낙상이 아닌 침수 | 상용 레이더는 낙상 감지. 본 시스템은 **물에 잠긴 상태**를 압력으로 감지 — 다른 물리 현상 |
| 생명 알림은 로컬 우선 | 인터넷이 끊겨도 동작해야 한다. Cloud·앱은 원격 조회 보조, 응급 경보는 로컬 |

## 분석 방법

현재는 **사전에 정의한 특징값(feature)** 기반의 통계적 분류. 시계열 딥러닝은 데이터 부족 시 과적합·설명 불가로 부적합하며, 데이터가 축적된 이후로 미룬다.

- 특징 F1~F6: 지속성 · 복귀 여부 · 복귀 시간 · 빈도 · 누적 dP · 단조성
- 공통 처리: 이동평균(평활화) → 1차 미분(변화율) → 임계값 + 지속시간(오탐 억제)

---

## 실험 대상

PHASE A에서는 **성인 남성 자발적 잠수를 우선**한다. 실제 인체는 부피·호흡·미세움직임을 재현하므로 모형보다 외적 타당성이 높고, 노인 욕조 침수 역학과도 정합한다.

> **안전 전제:** 자발적 잠수도 천수성 실신 위험이 있다. 감시자 상시 동석·얕은 수심·짧은 시간이 필수이며, 단독 실험은 금지한다. (RISKS.md Risk 14)

실제 익사 데이터는 윤리적으로 수집 불가능하므로, Phase A는 "실제 익사 감지"가 아니라 **"침수 변위 패턴의 분리 가능성"**을 검증한다.

## Phase A 판정 기준 (Exit Criteria, 확정)

충돌쌍(정상 vs 침수가 가장 헷갈리는 쌍: N4-D5, N2-D3, N5-D2)에서 판정한다.

| 게이트 | 지표 | 기준 | 비고 |
|---|---|---|---|
| 신호 존재 | SNR | **≥ 10 dB** | 정의 `20·log10(\|peak dP\| / σ)`, σ는 실측 노이즈로 확정 |
| 분리력 (핵심) | AUC | **≥ 0.90** | 충돌쌍별, 특징 F1~F6 통합 분류기, Leave-One-Trial-Out CV |
| 진단 | Cohen's d | 핵심 특징 **≥ 1.5** | 어떤 특징이 신호를 싣는지 + 과적합 감별 |
| 피벗 조건 | AUC | 어느 충돌쌍이라도 **< 0.75** | 압력 단독 분리 실패 |

> 두 정규분포 가정에서 `AUC = Φ(d/√2)`이므로 d와 AUC는 서로 다른 축(단일 특징 vs 통합 분류기)으로 분리해 사용한다. 원안의 `d≥0.8 AND AUC≥0.9`는 부정합(AUC≥0.9 ⇒ d≈1.81)이라 폐기했다.

## 선행연구 · 차별점

| 유형 | 선행 | 본 시스템과의 차이 |
|---|---|---|
| 특허 | **US9427115B2 — Bathtub monitors** (RJE International, 2016 등록 / 2024 소멸) — 욕조 내 피에조 압력센서로 움직임·무움직임을 감지, 일정 시간 무움직임 시 경보 | 움직임 유무 기반. 본 시스템은 수위 변위(dP)의 시간 패턴(복귀형/지속형)으로 침수를 분류 — 다른 신호·다른 판정 로직 |
| 특허 | **EP1918191A1** — 수압·호흡·자세 등을 이용한 욕조 익사 위험 감지 | 다중 센서 기반. 바닥 압력 변화 패턴 단독 분석과 다름 |
| 특허 | **US20150107015A1** — 영유아 욕조 익사 감지 및 자동 배수 | 모션 기반·영유아 대상. 압력 시간 패턴 방식이 아님 |
| 학술 | **Kharrat et al. (2012)** — 착용형 압력센서와 신경망을 이용하여 near-drowning과 정상 수영을 분류 | 착용형·수영장 대상. 욕조 고정 압력센서 기반 정상 입욕/침수 구분은 다루지 않음 |
| 학술 | **Murao et al.** — 욕조 바닥 수압센서를 이용한 사용자 식별 | 사용자 식별·점유 판단이 목적이며 정상 입욕/침수 분류는 아님 |
| 상용 | **Milesight VS373** — 60GHz 레이더 낙상 감지 | 비접촉 낙상 감지이나 침수(잠긴 상태)는 감지하지 않음 |

본 연구의 차별점은 **정상 입욕(복귀형)과 침수(지속형)를 바닥 압력 dP의 시간 패턴으로 구분하는 판별 과제·방법**이다. 대비되는 성격은 유효하다: **① 낙상이 아닌 침수, ② 움직임 유무가 아닌 시간 패턴, ③ 착용형이 아닌 욕조 고정.** dP 시간 패턴 판별과 직접 일치하는 선행은 확인 범위 내 미발견이나, "미발견 = 없음"이 아니며 정식 선행기술 조사가 별도로 필요하다.

## 시스템 흐름

센서가 압력을 재고, PC가 저장·분석한다.

```
[욕조]  MS5803 압력센서 (10Hz) + 온도 (1Hz)
          │  I2C
          ▼
       ESP32  ──[LittleFS 온칩 백업]
          │  WiFi / TCP 스트림
          ▼
   PC (logger.py)  (구조화 데이터 저장)
          │
          ▼
   특징 추출 (F1~F6)  →  통계 분류  →  판정 (SNR · AUC · Cohen's d)
```

응급 경보는 로컬 우선으로 설계하며, 통신·허브·Cloud·앱은 Phase A 통과 이후 단계다.

## 향후 확장 방향 (현재 연구 범위 제외)

Phase A 검증 되면, 입욕 침수 감지를 넘어 낙상·호흡·활동 패턴까지 통합하는 융합 안전 시스템으로 확장한다. 이 비전의 가설·한계는 [RISKS.md](./RISKS.md)에 `[장기 비전]` 태그로 별도 표기한다.
