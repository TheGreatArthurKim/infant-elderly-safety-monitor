# 압력 센서 작동 원리 — MS5803-14BA

> 이 문서는 Phase A의 물리적 전제를 정리한다.
> 데이터시트 수치는 **인용 기준**(TE Connectivity MS5803-14BA / DA5803-14BA_005)이며, 노이즈 σ 등 실측이 필요한 값은 별도 표기한다.

## 요약

**센서 위 물기둥 높이에 비례하는 절대압력**을 잰다. 측정 대상 물리량은 **수위(水位)의 시간 변화(dP)**이며, 침수는 그 시간 패턴으로 간접 추론한다.

---

## 1. 센서가 재는 것 — 절대압력

- MS5803-14BA는 **절대압(absolute)** 방식이다. **대기압을 포함**해 잰다. (인용 기준)
- 물속에서 읽는 값 = **대기압(≈1013 mbar) + 물기둥 압력(ρgh)**.
- baseline에는 항상 대기압이 깔려 있고, 물기둥은 그 위에 얹힌다.

내부 구조 (인용 기준):

- **압전저항(piezo-resistive) 소자 + 24bit ΔΣ ADC.** 압력이 다이어프램을 누르면 저항이 변하고, 이를 전압 → 디지털로 변환한다.
- 공장에서 두 온도·두 압력으로 개별 보정한 **계수 6개(C1~C6)를 128bit PROM에 저장.** 펌웨어가 이를 읽어 보정 공식에 써야 실제 mbar가 나온다.
- 원시값 두 개를 읽는다: **D1(압력)**, **D2(온도)**. D2로 D1을 온도 보정한다.
- **온도를 같이 재는 이유:** 압전저항은 온도에 민감해 보정 없이는 압력값이 틀어진다. 목욕물 온도는 100ms 안에 변하지 않으므로 **10Hz 압력 / 1Hz 온도** 구성으로 충분하다.

> 센싱 소자는 실리콘 겔 + 부식방지 스테인리스 캡으로 보호된다(인용 기준). 하우징에서 **센서 캡만 물에 노출**되면 되고, 나머지 전자부는 완전 밀폐해도 측정에 지장이 없다.

---

## 2. 압력 → 수위 변환

핵심 물리: **P = ρgh** (ρ=물 밀도, g=중력가속도, h=수위). 물에서 **1 mbar ≈ 물 10.2 mm ≈ 1 cm**.

| 사건 | 물리 | 센서가 보는 것 |
|---|---|---|
| 입수·전신 잠김 | 물 부피 밀어냄 → 수위 상승 | 압력 상승 (dP+) |
| 물장구·출렁임 | 순간 수면 요동, 평균 수위 유지 | dP 진동 후 baseline 복귀 (**복귀형**) |
| 침수 후 정지 | 배수량 유지 → 수위 유지 | dP 상승 후 유지 (**지속형**) |

**핵심 한계 (RISKS.md Risk 01의 물리 근거):** 센서는 "잠김"이 아니라 "수위 변화"만 안다. 이미 욕조에 앉아 목욕 중 **머리만 잠기면 배수 부피 변화 ≈ 0 → dP ≈ 0**. 압력 크기로는 이 케이스를 못 잡을 수 있으며, Phase A는 이것이 시간 패턴(복귀형 vs 지속형)으로 분리되는지를 검증한다.

수위 상승량 = 배수 부피 ÷ **욕조 단면적**. 좁은 욕조일수록 같은 배수량에 수위가 크게 오른다. 욕조 단면적이 통제변수인 이유다.

## 3. 분해능 vs 절대정확도

| 지표 | 값 (인용 기준) | 의미 |
|---|---|---|
| **분해능 (resolution)** | 0.2 mbar (OSR 4096) ≈ 물 **2 mm** | 감지 가능한 **최소 변화량** |
| **절대정확도 (absolute accuracy)** | 0~6 bar·0~40°C에서 −20 / +20 mbar ≈ 물 **±20 cm** | 절대값이 참값과 얼마나 벌어지나 |

dP는 절대값이 아니라 **상대 변화**다. 절대정확도의 오프셋 오차는 차분에서 상쇄되므로, dP 측정에 유효한 것은 **분해능과 노이즈뿐**이다. 절대정확도(±20 mbar)는 dP 측정 성능과 무관하다.

실제로 중요한 값은 **노이즈 σ** — SNR( `20·log10(|peak dP| / σ)` )의 분모다. 데이터시트에 없으므로 정지수(靜止水) baseline 구간에서 실측한다.

> 데이터시트류에서 "0.012 mbar"가 보이면 그건 **MS5803-01BA(1 bar 버전)의 압력 분해능**이거나 **14BA의 온도 분해능(0.012°C)**이다. 14BA 압력 분해능은 **0.2 mbar**다.

---

## 4. OSR / 샘플링 설정

- OSR(오버샘플링)이 높을수록 분해능↑, 변환시간↑. OSR 4096 변환시간 ≈ 8.2ms(전형)~9.0ms(최대), 한 변환당. (인용 기준)
- 압력 D1 + 온도 D2 ≈ 18ms. **10Hz(100ms 예산)에 여유** → OSR 4096(최고 분해능) 유지.
- F4(빈도, 2~4Hz 물장구)를 Nyquist로 잡으려면 10Hz 샘플링이 필요하며, OSR 설정과 충돌하지 않는다.

## 5. 측정 신뢰성 — YourCee(현재) vs SparkFun(최종)

| 항목 | 내용 |
|---|---|
| 칩 | 같은 MS5803-14BA면 **동작 원리·보정 공식 동일** |
| 실제 차이 | 브레이크아웃 PCB·디커플링·조립 품질·정품/문서화 |
| Phase A | 원리 검증은 YourCee로 충분 |
| 재현성 필요 단계 | 정품·문서화된 SparkFun로 재현 |
| 리스크 | 클론이 정품 TE 칩·정상 보정인지 불확실 |

**정품·보정 확인:** PROM에 **4bit CRC**가 있다(AN520). PROM 계수를 읽어 **CRC-4를 확인**하면 보정계수가 정상 개체인지 코드로 판별할 수 있다.

> 현재 펌웨어는 SparkFun MS5803 라이브러리(`SparkFun_MS5803_I2C.h`, 주소 `ADDRESS_HIGH` = 0x76)를 쓴다. 라이브러리는 SparkFun제, 하드웨어는 YourCee 클론이며, 정품 MS5803이면 정상 동작한다.

## 6. 물리 → 통제변수

Phase A 프로토콜의 통제변수는 위 물리에서 직접 도출된다.

| 물리적 사실 | 통제/기록 변수 | 이유 |
|---|---|---|
| 절대압(대기압 포함) | 대기압 로깅 또는 세션 짧게 | 장시간 baseline 드리프트 방지 |
| 수위상승 = 배수량 ÷ 단면적 | 욕조 단면적 (고정 또는 기록) | 신호 크기 좌우 |
| ρg 온도 의존 + 센서 온도보정 | 수온 (범위 고정·기록) | 밀도 ~0.6%(20↔40°C) + 보정 입력 |
| baseline = 초기 수위 | 시작 수심 고정·기록 | dP 기준선 |
| 배수량 = 신호원 | 피험자 몸무게·부피 | 신호 진폭 |
| 센서는 최저수위 아래여야 | 센서 위치·깊이·방향 고정 | 항상 물속 유지·재현성 |
| σ는 데이터시트에 없음 | 정지수 baseline 구간 확보 | SNR 분모 실측 |
| OSR–속도 트레이드 | 10Hz · OSR 4096 고정 | 조건 통일 |
